# Swift Client: Skill & Tool Detection and Rendering

This document explains how the Swift client can detect when the backend is executing a skill/tool, track execution progress, and render proper UI states and animations.

---

## **🔹 How Swift Knows A Skill Is Being Executed**

All skill execution flows through the Vercel AI SDK v7 stream protocol. The backend emits explicit structured events at every phase:

---

## **🔹 Stream Event Timeline**

This is the exact sequence of events sent over the SSE stream:

| Event Type | Description | When It Arrives |
|------------|-------------|-----------------|
| `start` | Stream initialized | Immediately when request starts |
| `skill-start` | ✅ **SKILL EXECUTION BEGIN** | Right before the skill is invoked |
| `text-start` | Text response has started | When skill returns first text chunk |
| `text-delta` | Individual text chunk | Repeated for every piece of output text |
| `text-end` | All text output complete | After last text chunk |
| `data-file_download` | File generated by skill | After text stream completes |
| `skill-complete` | ✅ **SKILL EXECUTION FINISHED** | After all processing including file downloads |
| `finish` | Full request completed | Final event in stream |

---

## **🔹 `skill-start` Event Payload**

**This is the event Swift should listen for to detect skill execution:**
```json
{
  "type": "data-skill_execution",
  "data": {
    "type": "skill_start",
    "slug": "potomac-docx-skill",
    "name": "Potomac DOCX Skill",
    "category": "document",
    "description": "Create professional Potomac-branded Word documents"
  }
}
```

This event is **guaranteed** to be sent **before any text output** when a skill is being executed. Regular LLM responses without skills will **never** send this event.

## **🔹 Swift State Machine Implementation**

Implement this state machine to properly track skill execution:

```swift
enum StreamState: Equatable {
    case idle
    case connecting
    case thinking
    case runningSkill(Skill)
    case streamingText
    case finalizing
    case complete([FileDownload], String)
    case failed(Error)
}

struct Skill: Decodable, Equatable {
    let slug: String
    let name: String
    let category: String
    let description: String
}

struct FileDownload: Decodable, Equatable {
    let file_id: String
    let filename: String
    let download_url: String
    let file_type: String
    let size_kb: Double
    let tool_name: String
}

@MainActor
class ChatStreamViewModel: ObservableObject {
    @Published var state: StreamState = .idle
    
    func handleEvent(_ event: StreamEvent) {
        switch event.type {
            
            // MARK: - Skill Detection Events
            case "data-skill_execution":
                if let skillData = event.data["data"] as? [String: Any],
                   skillData["type"] as? String == "skill_start",
                   let skill = try? JSONDecoder().decode(Skill.self, from: JSONSerialization.data(withJSONObject: skillData)) {
                    
                    // ✅ SKILL IS NOW RUNNING
                    state = .runningSkill(skill)
                    
                    // Log for debugging
                    print("Skill started: \(skill.name)")
                }
            
            // MARK: - Standard Stream Events
            case "start":
                state = .connecting
                
            case "text-start":
                state = .streamingText
                
            case "text-delta":
                // Append text to message buffer
                if case .streamingText = state {
                    // Update text view
                }
                
            case "text-end":
                state = .finalizing
                
            case "data-file_download":
                if let file = try? JSONDecoder().decode(FileDownload.self, from: event.data) {
                    // Store file reference for UI
                }
                
            case "skill-complete":
                // Skill has finished all processing including file downloads
                state = .complete(files, fullText)
                
            case "finish":
                if case .finalizing = state {
                    state = .complete([], fullText)
                }
                
            case "error":
                state = .failed(event.error)
        }
    }
}
```

## **🔹 How To Render Different Skill States**

| State | UI Rendering Behaviour | Animation |
|-------|-------------------------|-----------|
| **`.connecting`** | Show standard chat typing indicator | Pulsing animation |
| **`.thinking`** | Show generic LLM processing indicator | Bouncing pulsing dots |
| **✅ `.runningSkill(skill)`** | **This is the important state:** <br>1.  Show skill-specific header with skill name and icon <br>2.  Display skill description <br>3.  Show progress indicator with skill category animation <br>4.  **Do NOT show regular typing indicator** | Skill-specific loading animation: <br>• `document` category: page build animation <br>• `presentation` category: slide animation <br>• `spreadsheet` category: table animation <br>• All others: generic progress spinner |
| **`.streamingText`** | Show skill output streaming word-by-word | Smooth text fade-in animation |
| **`.finalizing`** | Show "Preparing download..." message with success indicator | Checkmark animation |
| **`.complete`** | Render full text response + file download card(s) with bounce-in animation | Bounce entrance for file cards |

## **🔹 Critical Rules For Swift Client**

1.  **Always prefer `skill-start` event over text matching:**
    - Never try to detect skills by looking for keywords in the text
    - The `skill-start` event is authoritative and will arrive **before** any text
    - Text content can change - the event will always be sent exactly the same way

2.  **Skills run even with no text output:**
    - Some skills may produce only a file with no accompanying text
    - You may receive `skill-start` followed directly by `data-file_download` with no text events in between
    - Never assume text will be present

3.  **All skills follow exactly the same pattern:**
    - potomac-docx-skill
    - potomac-pptx
    - potomac-xlsx
    - backtest-expert
    - quant-analyst
    - financial-deep-research
    - Every future skill will automatically send these events

4.  **Skills can run for a long time:**
    - Document generation can take 10-60 seconds
    - Do not time out the stream
    - Show appropriate progress and keep UI responsive
